# 晚秋记账 — 项目说明

## 项目概述

"晚秋记账"是一款运行在 Windows 上的个人记账应用，用于记录人民币花销并支持分类管理。

## 用户背景

用户是技术小白，不懂编程。在整个项目开发周期内，**必须严格遵守以下规则**：

### 规则1：列出方案，让用户决策
遇到任何技术决策时（选框架、选库、选架构等），必须：
- 列出至少 2-3 个可行方案
- 对每个方案用通俗语言解释其优缺点和适用场景
- 让用户根据自己的需求做出选择
- **绝对不能替用户做技术决策**

### 规则2：每个技术点都要详细解释
涉及任何技术术语、概念、代码、配置文件时，必须：
- 遇到技术术语时，给出通俗的解释

### 规则3：用户无法提供具体技术需求
用户只能描述功能层面的需求（比如"我想记录花销"），无法提供技术层面的需求（比如"用什么数据库"）。所有技术层面的需求必须由我来设计方案并列出选项，让用户选择。

## 产品功能

### 核心功能
1. **记录花销**：输入金额、选择分类、填写备注（可选）、选择日期
2. **账单列表**：按时间倒序展示，支持按月份筛选
3. **分类管理**：预设分类 + 用户自定义
4. **月度统计**：饼图展示各分类支出占比
5. **本地存储**：数据保存在本地，关闭后不丢失

### 预设分类
餐饮、交通、住房、服饰、医疗、通讯、娱乐、学习、人情、工作、收入、其他（共12个大类）

### 数据字段
id、amount（金额）、type（支出/收入）、category（分类）、subCategory（子分类）、note（备注）、date（日期）

## 技术方案（已选定）

**方案：Python + PyQt6 + SQLite + Matplotlib**

> 注：最初选择了 Electron 方案，但在 Windows 上遇到 Electron 底层兼容性 Bug（`require('electron')` 无法正常工作），经多次尝试（Electron 43 → Electron 28 → 多种 workaround）均无法解决。最终换用 Python 方案，安装快、无兼容问题。

| 组件 | 用途 | 通俗解释 |
|------|------|---------|
| Python | 编程语言 | 语法简单，Windows 系统自带或安装便捷 |
| PyQt6 | 桌面界面库 | 提供窗口、按钮、表格等桌面控件 |
| SQLite | 数据库 | Python 内置，不需要额外安装，数据保存在本地 |
| Matplotlib | 图表库 | 在统计页绘制饼图 |

### 运行方法

```bash
# 安装依赖（只需执行一次）
pip install PyQt6 matplotlib -i https://pypi.tuna.tsinghua.edu.cn/simple

# 启动应用
python main.py
```
